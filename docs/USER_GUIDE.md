# GroGoliath User Guide

## Overview
GroGoliath helps you turn CSV data into SEO pages using templates and AI.
The app has five main areas: Dashboard, Projects, Datasets, Templates, and Settings.

## Quick Start
1. Go to Datasets and upload a CSV file.
2. Create a new Project from a CSV or from a saved Dataset.
3. Open the project and click Generate to create pages.
4. Review results in View Data and export CSV when ready.

## Dashboard
- Shows totals for projects and keywords.
- Use New Project to start a new campaign.

## Projects
- Search and filter by status (Draft or Completed).
- Actions per project:
  - Generate: run AI generation
  - Edit: rename project or change platform
  - View: open the data table
  - Download: export CSV
  - Delete: remove the project

## New Project (Modal)
- Choose a platform: Wordpress, Webflow, or Shopify.
- Choose one of three creation modes:
  - Upload CSV: import a new file
  - From Library: use a saved Dataset
  - Manual: create an empty project

## Generate (AI)
- Pick a template or write a custom prompt.
- Choose the target output column (default is AI_Output).
- Start generation and monitor progress in the log.
- You can stop the run at any time.

## View Data
- Edit cells directly in the table.
- Add rows or new columns when needed.
- HTML cells can be opened with Inspect for preview and safe editing.
- Export the data to CSV when ready.

## Datasets
- Upload CSV files once and reuse them across projects.
- Delete datasets you no longer need.

## Templates
- Create and edit templates in the Template Builder.
- Preview templates before using them in generation.
- Use preset templates as starting points.

## Template Builder
- Add blocks like Hero, Text, Pricing, FAQ, and more.
- Use Columns (2) or Grid (2x2) for responsive layouts.
- Click the help icon on any block to learn what it does.
- Use variables in the format {{ColumnName}} to insert CSV values.

## Settings
- View your account email.
- Sign out.

## Tips
- Keep CSV headers simple and consistent.
- Test a small dataset first before large runs.
- Use Templates to keep output structure consistent across many pages.
